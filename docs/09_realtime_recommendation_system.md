# 초개인화 실시간 추천 시스템 구현 보고서 (09)

추천 페이지의 데이터 미표시 문제를 해결하고, 사용자의 현재 위치, 실시간 날씨, 시간을 완벽하게 연동하는 지능형 추천 엔진을 구축했습니다.

## 1. 주요 해결 내용

### 1.1 추천 데이터 누락 이슈 해결
- **문제**: API 호출 과정의 지연 또는 오류 시 빈 화면이 나타나는 현상.
- **해결**: API 응답이 없거나 네트워크가 불안정한 경우에도 즉시 준비된 **로컬 고화질 추천 데이터(MOCK)**를 활성화하여 끊김 없는 사용자 경험을 보장했습니다.

### 1.2 위치 정보 고정 문제 해결 (진짜 내 주변 추천)
- **문제**: 위치 권한 승인 후에도 '강남' 지역에만 한정된 추천이 제공됨.
- **해결**:
  - `App.tsx`에서 취득한 실제 위경도 좌표를 추천 엔진으로 직접 전달합니다.
  - 백엔드 검색 쿼리에서 "주변" 키워드를 동적으로 결합하여, 사용자가 있는 실제 위치 기반의 Naver 검색 결과를 가져오도록 개선했습니다.

## 2. 지능형 상황 인지(Context-aware) 추천 로직

타 서비스와 차별화되는 **상황 인지 기능**을 엔진에 탑재했습니다.

### ☀️ 날씨 연동 (Real-time Weather)
- **비/흐림**: "비 오는 날 어울리는 뜨끈한 국물 요리" 큐레이팅 (예: 짬뽕, 칼국수, 국밥 등).
- **눈**: 따뜻한 전골 및 나베류 우선 추천.
- **맑음**: 사용자 평소 선호도 및 건강 목표(다이어트/벌크업)에 집중.

### ⏰ 시간 연동 (Time-based Curation)
- **아침 (05~10시)**: 샌드위치, 샐러드, 가벼운 브런치 위주.
- **점심/저녁**: 활동량에 맞춘 든든한 일반 식사류.
- **심야 (21~02시)**: 부담 없는 야식 및 24시간 운영 식당 필터링 강화.

## 3. 기술적 스택 및 데이터 흐름
- **Frontend**: 
  - `navigator.geolocation` + `Open-Meteo API` (실시간 기상 정보).
  - `framer-motion` (카드 스택 UI 동기화).
- **Backend (FastAPI)**: 
  - 확장된 `/recommend` 엔드포인트를 통해 환경 변수 수집.
  - 유동적인 검색어 조합 엔진을 통한 Naver Local API 호출.

이로써 **Tastemate**는 단순한 검색 앱을 넘어, 사용자의 환경을 완벽하게 이해하고 제안하는 진정한 **밥친구**로 거듭났습니다.
