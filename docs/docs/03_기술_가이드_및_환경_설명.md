# [보고서 03] 상세 기술 가이드 및 개발 환경 명세 (Part 4)

본 문서는 **밥친구(Ricefreind)** 서비스의 세부 코드 구조, 개발 환경 설정, 그리고 API 연동 방식에 대한 상세한 기술적 정보를 제공합니다. 개발자 온보딩 및 시스템 고도화의 기초 자료로 활용됩니다.

---

## 💻 1. 개발 환경 명세 (Development Environment)

### A. 런타임 및 운영체제
- **OS**: Windows (시스템 환경 기준)
- **Backend**: Python 3.10 이상
- **Frontend**: Node.js 18.x / 20.x 이상 (Vite 6 사용)

### B. 주요 의존성 (Key Dependencies)
- **백엔드 (Backend)**:
  - `fastapi`: API 프레임워크
  - `uvicorn`: ASGI 서버
  - `pydantic`: 데이터 검증 및 스키마
  - `python-dotenv`: 환경 변수 관리
  - `openai`: LLM(RAG) 연동
- **프론트엔드 (Frontend)**:
  - `react`: UI 라이브러리 (v18.3)
  - `vite`: 빌드 도구 및 개발 서버 (v6.3)
  - `tailwindcss`: 스타일링
  - `lucide-react`: 아이콘 에셋
  - `framer-motion`: 애니메이션 효과

---

## 📂 2. 주요 코드 구조 분석 (Codebase Walkthrough)

### 🐍 Backend Structure
- `/backend/main.py`: FastAPI의 진입점으로 CORS 설정, 미들웨어, 라우터 포함을 담당합니다.
- `/backend/api/router.py`: 모든 도메인별 라우터를 하나의 API 경로로 결합합니다.
- `/backend/api/endpoints/`:
  - `meals.py`: 식사 기록 CRUD (Supabase 연동 완료).
  - `recommend.py`: 사용자의 영양 상태에 따른 상세 식단 추천 (Framer Motion UI 대응 모델).
  - `profile.py`: 사용자 신체 정보 및 목표 설정 관리 (Supabase 연동 완료).
- `/backend/core/`: RAG 엔진(`rag.py`) 등 핵심 비즈니스 로직 및 AI 처리 모델 포함.
- `/backend/models/`: 추천 랭커(`ranker.py`)와 같은 ML/수학적 모델 파트.

### ⚛️ Frontend Structure
- `/frontend/src/App.tsx`: 최상위 컴포넌트로 화면 전환(Screen State) 및 글로벌 상태 관리.
- `/frontend/src/components/`:
  - `ChatbotScreen.tsx`: AI 상담 기능과 Supabase Edge Functions 연동.
  - `DashboardHome.tsx`: 메인 화면 대시보드 및 요약 정보 표시.
  - `MealLogScreen.tsx`: 식단 입력 및 칼로리 추적 로직.
- `/frontend/src/utils/`: Supabase 클라이언트 및 전역 설정값 관리.

## 🔌 3. API 및 외부 서비스 연동 가이드

### A. API 엔드포인트 요약 (Base URL: `http://localhost:8000/api/v1`)
| Endpoint | Method | Description |
| :--- | :--- | :--- |
| `/recommend/{user_id}` | GET | AI 기반 개인 맞춤 식단 추천 |
| `/meals/` | POST | 사용자의 식사 내역 서버 기록 |
| `/profile/{user_id}` | GET/PATCH | 사용자 프로필 정보 조회/수정 |

### B. 외부 API 통합 방식
1. **Kakao Login & Map**:
   - 프론트엔드에서 JavaScript SDK를 로드하여 인증 및 지도 렌더링 처리.
   - 백엔드에서 `KAKAO_REST_API_KEY`를 사용하여 추가적인 사용자 정보를 획득(Part 3 예정).
2. **OpenAI (RAG)**:
   - `core/rag.py`에서 사용자의 질문과 관련 컨텍스트를 OpenAI 모델에 전송하여 개인화된 답변 생성.
3. **Supabase Integration**:
   - `getSupabaseConfig()` 유틸리티를 통해 인증 키를 관리하며, Edge Functions를 호출하여 서버리스 로직 수행.

## 🛠️ 4. 개발 및 디버깅 팁

- **로그 확인**: 백엔드 실행 시 `--reload` 옵션을 사용하여 코드 변경 사항을 즉시 반영하고, 터미널 로그를 통해 API 요청/응답을 모니터링합니다.
- **환경 변수**: 신규 개발자 합류 시 반드시 `.env.example`을 기반으로 유효한 API 키가 포함된 `.env` 파일을 생성해야 합니다.
- **포트 충돌 주의**: 프론트엔드(5173), 백엔드(8000) 포트가 이미 사용 중인지 확인하세요.
