# [보고서 04] 식단 추천 UX 리팩토링 계획 (스와이프 UI 도입)

본 보고서는 사용자의 요청에 따라 현재의 리스트/단일 카드 방식인 식단 추천 UI를 틴더(Tinder) 스타일의 **스와이프 기반 인터페이스**로 리팩토링하기 위한 상세 계획을 담고 있습니다.

---

## 🎨 1. 디자인 목표 (Design Objective)

- **몰입감 있는 시각적 경험**: 이미지를 화면의 60~70% 이상 차지하게 배치하여 시각적 즐거움을 제공.
- **직관적인 제스처**: Framer Motion을 활용하여 부드러운 스와이프(좌: 거절, 우: 선택) 감각 구현.
- **점진적 정보 노출 (Progressive Disclosure)**:
    - **기본 뷰**: 메뉴 사진 + 식당 이름 + 핵심 메뉴 + 거리.
    - **상세 뷰**: 카드 클릭 시 하단에서 슬라이드 업 하거나 카드가 확장되며 상세 정보(가격, 추천 사유, 영양 성분 등) 노출.

---

## 🛠️ 2. 기술적 구현 전략 (Technical strategy)

### A. Framer Motion 활용
- `AnimatePresence`를 사용하여 카드가 사라지고 나타날 때의 애니메이션 처리.
- `useMotionValue`, `useTransform`을 이용해 스와이프 거리에 따른 카드 회전(Rotation) 및 색상 피드백(빨강/초록) 구현.

### B. 컴포넌트 구조 재설계
- `SwipeableCardStack`: 카드 뭉치를 관리.
- `SwipeableCard`: 개별 카드의 애니메이션 및 제스처 로직 담당.
- `RestaurantDetailOverlay`: 선택된 식당의 상세 정보를 보여주는 바텀 시트 또는 모달.

### C. 사용자 액션 버튼
- 하단에 큼직한 **X (Skip)** 버튼과 **❤️ (Like)** 버튼 배치.
- 중앙에는 상세 정보를 볼 수 있는 **ℹ️ (Info)** 버튼 추가.

---

## 📈 3. 단계별 로드맵 (Step-by-step Roadmap)

| 단계 | 작업 내용 | 우선순위 |
| :--- | :--- | :--- |
| **Step 1** | 기존 코드 백업 및 Framer Motion 기반 카드 스택 프로토타입 구현 | 높음 |
| **Step 2** | 스와이프 제스처 로직 구현 (좌: Dislike, 우: Like) | 높음 |
| **Step 3** | 상세 정보 보기 레이아웃 및 애니메이션 추가 | 중간 |
| **Step 4** | 실제 데이터와 연동 및 모바일 반응형 최종 튜닝 | 중간 |

---

## ⚠️ 4. 리스크 관리 (Risk Management)

- **성능 저하**: 애니메이션이 과도할 경우 저사양 모바일 기기에서 버벅임이 발생할 수 있음 -> `layout` 프로퍼티 최적화.
- **복구 대책**: 현재 작업 시작 전 `src/backup_recommendation` 폴더에 원본 코드를 백업 완료함. 문제 발생 시 즉시 복구 가능.
