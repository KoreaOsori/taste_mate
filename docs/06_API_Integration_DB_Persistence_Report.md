# API 연동 및 DB 데이터 영속화 보고서

본 보고서는 온보딩 시 수집된 데이터를 백엔드 API와 Supabase DB에 안정적으로 저장하고, 이를 통해 개인화된 서비스를 제공하기 위한 연동 작업 내용을 기록합니다.

## 1. 개요
현재 TasteMate는 온보딩 시 다양한 사용자 정보를 수집하며, 이 데이터는 사용자의 영양 분석 및 식당 추천 알고리즘의 핵심 입력값으로 활용됩니다. 데이터의 유실을 방지하고 연속적인 사용자 경험을 제공하기 위해 DB 영속화 로직을 구현했습니다.

## 2. 주요 연동 내역

### 백엔드 (API & DB)
- **UserProfile 모델 확장**: 프런트엔드에서 수집하는 모든 필드(기피 음식, 식사 시간 등)를 수용할 수 있도록 Pydantic 모델 및 DB 처리 로직을 보완했습니다.
- **데이터 매핑 최적화**: 프런트엔드의 `camelCase` 필드들을 백엔드의 `snake_case` 데이터베이스 스키마와 상호 호환되도록 처리했습니다.

### 프런트엔드 (Integration)
- **온보딩 저장 로직 고도화**: 온보딩 완료 시 `profileService.updateProfile`을 호출하여 Supabase DB(`profiles` 테이블)에 실시간으로 정보를 저장합니다.
- **사용자 식별**: `crypto.randomUUID()`를 통해 고유 사용자 ID를 생성하고, 이를 세션 동안 유지하여 데이터 무결성을 보장합니다.

## 3. 구현 상세
| 항목 | 상세 내용 |
| :--- | :--- |
| **API 엔드포인트** | `PUT /api/v1/profile/{user_id}` |
| **DB 테이블** | `profiles` (Supabase) |
| **주요 필드** | `disliked_foods`, `breakfast_time`, `activity_level`, `goal` 등 |

## 4. 향후 과제
- **로그인 세션 유지**: 브라우저 로컬 스토리지를 활용하여 페이지 새로고침 시에도 기존 유저 정보를 자동으로 불러오는 로직 강화.
- **동적 설정**: `.env`에 정의된 API 키와 프롬프트 설정값을 백엔드에서 동적으로 로드하여 하드코딩된 메세지들을 대체.
