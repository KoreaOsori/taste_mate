# [보고서 20] 클라우드 배포 가이드 (Vercel & Railway)

본 문서는 로컬 개발 환경에서 완성된 **밥친구(Ricefreind)** 서비스를 외부 URL로 배포하기 위한 상세 절차를 제공합니다.

---

## 🚀 1. 백엔드 배포 (Railway)
Railway는 Dockerfile을 자동으로 감지하여 빌드 및 배포를 수행하는 최적의 서비스입니다.

1.  **[Railway.app](https://railway.app/)** 접속 및 GitHub 계정으로 로그인.
2.  **New Project** -> **Deploy from GitHub repo** 선택.
3.  `taste_mate` 저장소 선택 후, **Root Directory**를 `backend`로 설정.
4.  **Variables** 탭에서 다음 환경 변수 등록:
    - `NAVER_CLIENT_ID`, `NAVER_CLIENT_SECRET`
    - `KAKAO_REST_API_KEY`, `KAKAO_NATIVE_APP_KEY`
    - `SUPABASE_URL`, `SUPABASE_ANON_KEY`
    - `OPENAI_API_KEY`
    - `FRONTEND_URL`: (이후 Vercel에서 발급받은 URL 입력)
5.  **Settings** -> **Public Networking**에서 도메인 생성 (예: `tastemate-api.up.railway.app`).

---

## 🎨 2. 프론트엔드 배포 (Vercel)
Vercel은 React(Vite) 앱의 빌드 및 배포를 자동화하며 최상의 성능을 보장합니다.

1.  **[Vercel.com](https://vercel.com/)** 접속 및 GitHub 계정으로 로그인.
2.  **Add New** -> **Project** 선택 후 저장소 Import.
3.  **Framework Preset**: `Vite` 선택.
4.  **Root Directory**: `frontend` 선택.
5.  **Environment Variables** 등록:
    - `VITE_API_BASE_URL`: (위 Railway에서 생성한 도메인 + `/api/v1`)
    - `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`
    - `VITE_KAKAO_JAVASCRIPT_KEY`
    - `VITE_USE_PROXY`: `false`
6.  **Deploy** 클릭 후 발급된 URL(예: `ricefriend.vercel.app`) 확인.

---

## 🔐 3. 외부 서비스 설정 (CORS & Auth)
배포된 URL이 정상 동작하려면 외부 서비스의 보안 설정을 업데이트해야 합니다.

- **Supabase**: `Authentication` -> `Redirect URLs`에 Vercel 주소 추가.
- **Kakao Developers**: `내 애플리케이션` -> `플랫폼` -> `Web`에 Vercel 주소 추가.
- **Railway**: 백엔드의 `FRONTEND_URL` 변수에 Vercel 주소를 입력하여 배포 재시작.

---

## ✅ 4. 최종 확인
