# [보고서 11] TasteMate 현재 코드 현황, 개발 상황 및 향후 로드맵

본 문서는 **TasteMate** 프로젝트 단일 환경에 집중하여, 현재까지의 개발 진행 상황과 코드베이스 구조, 그리고 향후 고도화 로드맵을 종합적으로 정리한 보고서입니다.

## 🏗️ 1. 현재 코드베이스 구조 (Codebase Overview)

TasteMate는 클라우드 BaaS(Supabase)를 중심으로 백엔드(FastAPI)와 프론트엔드(React/Vite)가 분리되어 통신하는 모던 웹 애플리케이션 구조를 갖추고 있습니다.

### A. 백엔드 (Backend - FastAPI)
**위치**: `taste_mate/backend/`
- `main.py`: 서버 진입점, CORS 정책 관리 및 API 라우터 연결
- `api/`: 인증(`auth`), 사용자 프로필(`profile`), 식단 추천(`recommend`), 식사 기록(`meals`) 등 도메인별 라우터 분리
- `core/`: 향후 고도화될 AI(RAG) 엔진 및 공통 비즈니스 로직
- `db/`: Supabase PostgreSQL 통신 클라이언트 및 쿼리 최적화
- **핵심 역량**: Python 3.10+ 기반으로 안정적이고 빠른 API 서비스 제공 및 Supabase Admin SDK를 통한 인증/DB 제어

### B. 프론트엔드 (Frontend - React + Vite)
**위치**: `taste_mate/frontend/`
- `src/App.tsx`: 전역 상태 관리 및 라우팅 흐름 제어, 앱 초기화(Auth 체크 등)
- `src/components/`:
  - `LoginScreen`, `OnboardingScreen`: 사용자 인증 및 초기 프로필 수집 로직
  - `HomeScreenNew`, `RestaurantRecommendationScreenNew`: 실시간 위치 연동, 스와이프 인터페이스(Framer Motion) 기반의 메인 UI
  - `ProfileScreen`: Supabase 연동 프로필 조회 및 수정
- `src/utils/`: Supabase 클라이언트 연동(`supabaseClient.ts`), 전역 API 호출(`apiClient.ts`) 관리
- **핵심 역량**: TailwindCSS와 Framer Motion을 결합한 유려한 UX 제공 및 실시간 주소 동기화(카카오 API)

## ✅ 2. 개발 진행 현황 (Current Progress)

현재까지 **Part 1 ~ Part 3**에 이르는 주요 핵심 기능들의 뼈대와 실 데이터 연동이 완료되었습니다.

1. **프론트엔드-백엔드 통합 및 도커화**
   - 통합 API 통신 환경 구축 및 `docker-compose`를 이용한 로컬 컨테이너 실행 환경 완비
2. **UX 고도화 및 실시간 데이터 처리**
   - **스와이프 UI**: 틴더 스타일의 지루하지 않은 식단 평가 및 추천 인터페이스 적용
   - **실시간 주소 연동**: 사용자 좌표 기반 카카오 Reverse Geocoding을 통해 현재 화면에 실시간 위치 텍스트 반영
3. **데이터베이스 안정화 및 로그인 구조 개선**
   - **Supabase 도입**: 기존 Mock 데이터를 폐기하고 실제 PostgreSQL DB에 `Profiles` 등 테이블 연동 완료
   - **인증 파이프라인**: Supabase Auth와 백엔드 API를 결합하여 로그인/회원가입의 명확한 에러 핸들링(한국어 메시지, 가입 유무 체크) 및 온보딩 직행 로직 구축 완료

## 🚀 3. 향후 개발 로드맵 (Future Roadmap)

현재 안정화된 기초 인프라를 바탕으로, 추천 엔진의 지능화와 서비스 확장에 초점을 맞출 예정입니다.

### [Phase 1] 추천 알고리즘 고도화 및 데이터 확장
- **음식 및 식당 DB 연동**: `foods_v2` 및 `recommendation_history` 등 추천에 쓰일 핵심 데이터 테이블 스키마 정립 및 실제 데이터 파이프라인 구축
- **하이브리드 추천 RAG**: 기존 단순 추천을 넘어, 사용자의 선호/비선호 음식과 현재 날씨 정보를 Vector DB와 결합한 LLM 맞춤형 챗봇 및 응답 로직 개발

### [Phase 2] 플랫폼 기능 정교화
- **소셜 로그인 완성**: 현재 구축된 Supabase 기반 위에 카카오/네이버 등 OAuth 모듈을 추가하여 가입 허들 최소화
- **이미지 업로드 스토리지**: 사용자 프로필 이미지 및 음식 리뷰 인증샷 구현을 위한 Supabase Storage 서비스 연동

### [Phase 3] 서비스 운영 환경 구축
- **CI/CD 파이프라인**: 깃허브 액션(GitHub Actions)을 활용하여 프론트/백엔드 자동 빌드 및 클라우드 배포(Vercel, Render 등) 파이프라인 구축
- **성능 및 비용 최적화**: API 외부 호출(카카오 맵/OpenAI 등) 캐싱 로직 추가 및 Edge Functions를 통한 트래픽 분산 처리
