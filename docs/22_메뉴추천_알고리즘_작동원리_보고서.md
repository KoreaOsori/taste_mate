# 밥친구(Taste Mate) 서비스: 메뉴 추천 로직 및 알고리즘 보고서

본 보고서는 '밥친구' 서비스의 핵심 가치인 '개인화된 메뉴 추천'이 어떠한 기술적 메커니즘을 통해 이루어지는지 상세히 설명합니다. 본 서비스는 **3단계 하이브리드 추천 아키텍처(3-Phase Hybrid Architecture)**를 채택하여 정확도와 감성적 만족도를 동시에 확보하였습니다.

---

## 🏗️ 1. 하이브리드 추천 아키텍처 개요

단순히 랜덤하게 음식을 보여주는 것이 아니라, 사용자의 **정적 프로필, 동적 상태, 실시간 환경 데이터**를 결합하여 추천합니다.

1.  **Phase 1: 결정론적 세그먼트 필터 (Deterministic Segment Filtering)**
    - 사용자의 명시적 답변(예산, 동행, 종류 등)을 기반으로 후보군을 1차 선별합니다.
2.  **Phase 2: 확률적 AI 개인화 (Probabilistic AI Personalization)**
    - 선별된 후보군 중 사용자의 현재 상황(날씨, 기분, 시간, 잔여 칼로리)에 가장 적합한 'Best 5'를 LLM이 추론합니다.
3.  **Phase 3: 실시간 인프라 매핑 (Real-time Infrastructure Mapping)**
    - 추론된 메뉴를 바탕으로 실제 주변에 영업 중인 식당 정보를 API를 통해 결합합니다.

---

## 🔍 2. 단계별 상세 작동 원리

### [Phase 1] Segment Filter (전문가 시스템)
사용자가 설문(Q&A)에서 응답한 내용을 바탕으로 `foods_v2` 데이터베이스(약 500개 이상의 메뉴 데이터셋)에서 후보군을 추출합니다.

-   **Meal Role 필터**: 질문 시작 시 '본 식사'인지 '디저트/커피'인지에 따라 데이터셋의 대분류를 분리합니다.
-   **Hard Constraints**: 사용자가 선택한 **음식 종류(Category)**, **예산(Budget)**, **동행(Companion)** 정보를 필드로 매칭하여 부적절한 후보를 제거합니다.
-   **Soft Fallback**: 필터링 결과가 너무 적을 경우(3개 미만), 조건을 단계적으로 완화하여 사용자에게 항상 결과가 노출되도록 설계되었습니다.

### [Phase 2] AI Personalization (LLM Thinking)
1단계에서 살아남은 후보군 데이터와 사용자의 비정형 상태 데이터를 **GPT-4o-mini** 모델에게 전달합니다.

-   **Context Injection**: 현재 기온/날씨, 현재 시간(점심/저녁/야식), 사용자의 기분, 오늘 이미 섭취한 칼로리를 프롬프트에 주입합니다.
-   **Reasoning**: AI는 단순 메뉴 선정을 넘어, "비 오는 날의 적막함을 달래줄 따뜻한 국물 요리"와 같은 **감성적 추천 사유(Reason)**를 메뉴별로 생성합니다.
-   **Output**: 5개의 최종 메뉴와 각 메뉴에 대한 맞춤형 조언을 JSON 형태로 반환합니다.

### [Phase 3] Enrichment Engine (실시간 데이터 결합)
AI가 추천한 메뉴 이름(예: '마라탕')을 실생활과 연결하는 단계입니다.

-   **Spatial Search (Naver Local API)**: 사용자 주변(또는 설정 지역)의 실제 맛집을 검색합니다. 이때 대형 프랜차이즈나 관련 없는 상호를 필터링하는 **브랜드 필터링** 로직이 작동합니다.
-   **Geo-Fencing (Kakao Map API)**: 식당의 실제 위경도 좌표를 획득하고, 사용자와의 직선 거리를 계산하여 도보 이동 가능 여부를 판단합니다.
-   **Visual Enrichment (Naver/Google API)**: 해당 식당의 최신 블로그 이미지나 방문자 리뷰 사진을 병렬로 검색하여 사용자에게 시각적 즐거움을 제공합니다.
-   **Operational Data**: 식당별 영업시간(Business Hours)을 휴리스틱 알고리즘으로 분석하여 현재 방문 가능 여부를 유추합니다.

---

## 💡 3. 핵심 기술적 차별점

1.  **RAG(Retrieval-Augmented Generation) 패턴의 적용**: 외부 지식(음식 데이터셋)과 LLM의 추론 능력을 결합하여 환각 현상을 방지하고 정확한 정보를 제공합니다.
2.  **병렬 처리(Asynchronous Processing)**: 이미지 검색, 좌표 변환, 식당 검색을 동시에 수행하여 복잡한 로직임에도 불구하고 2~3초 내외의 빠른 응답 속도를 유지합니다.
3.  **영양학적 고려**: 사용자의 권장 섭취량 대비 현재 잔여 칼로리를 계산하여, 과도한 고칼로리 메뉴는 후순위로 밀어내는 건강 중심의 로직이 내장되어 있습니다.
