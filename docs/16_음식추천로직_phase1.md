# 🍱 음식 추천 로직 및 UI 고도화 (Phase 1)

본 문서는 실시간 데이터(위치, 날씨, 신체 데이터)와 LLM을 결합하여 초개인화된 맛집 추천 서비스를 구축한 **Phase 1**의 핵심 기술 내용과 구현 성과를 기록합니다.

---

## 1. 개요 및 목표
단순한 메뉴 추천을 넘어, 사용자의 **'현재 상황'**과 **'실제 주변 맛집 정보'**를 완벽하게 결합하여 신뢰성 높은 추천 서비스를 제공하는 것을 목표로 합니다.

- **위치 기반 정밀도**: 사용자의 현재 동네(행정동 단위)를 검색어에 반영.
- **데이터 일관성**: 추천된 메뉴와 검색된 식당 간의 1:1 매칭 보장.
- **시각적 완성도**: 엉뚱한 이미지를 배제하고, 어떤 사진 위에서도 글자가 잘 보이는 고대비 UI 구현.

---

## 2. 주요 핵심 기술 (Key Technical Achievements)

### 📍 행정동 기반 정밀 위치 검색 (Location-Aware Search)
- **기술**: Kakao Local API (Reverse Geocoding) 연동.
- **내용**: 사용자의 좌표(`lat`, `lng`)를 "역삼동", "송도동" 등의 행정구역 명칭으로 변환하여 네이버 검색어에 포함.
- **효과**: 네이버가 서버 위치나 임의 지역 결과를 반환하는 문제를 해결하고, 실제 사용자의 생활권 내 식당을 우선적으로 노출.
- **위치 없을 때**: 기본값(강남)으로 검색하지 않음. `use_location=false` 로 호출하면 **지역 키워드 없이 메뉴명만 검색**하여 "비슷한 맛집"만 추천. 상세는 `docs/17_추천_로직_상세_및_위치정책.md` 참고.

### 🍱 메뉴-식당 1:1 정밀 매칭 (Per-Menu Targeted Discovery)
- **문제**: 기존의 광범위한 지역 검색은 추천된 메뉴(예: 제육볶음)와 검색된 식당(예: 샐러드 집)이 맞지 않는 불일치 발생.
- **해결**: LLM이 선정한 추천 메뉴별로 **개별 독립 검색**을 수행하도록 로직 개편.
- **효과**: "제육볶음"이 추천되면 반드시 제육볶음 전문점 혹은 해당 메뉴를 취급하는 식당만 노출되는 100% 신뢰성 확보.

### 🖼️ 지능형 이미지 폴백 시스템 (Smart Image Relevance)
- **로직**: 다단계 검색 프로세스 도입.
  1. `[식당명] [메뉴명]` 검색 (정밀 매칭)
  2. 실패 시 `[식당명] 대표메뉴` 검색
  3. 실패 시 해당 **카테고리(한식, 일식, 카페 등) 고해상도 테마 이미지** 즉시 매핑.
- **효과**: 이미지 없음(Broken Image)이나 엉뚱한 사진 노출을 원천 차단하여 서비스 시각적 품질 유지.

## 3. UI/UX 디자인 고도화 (Premium Interface)

### 📱 스와이프형 카드 UI (Tinder-style UX)
- `framer-motion`과 `react-tinder-card`를 활용하여 직관적이고 재미있는 탐색 환경 구축.
- **Robust Layout**: Flex 컨테이너 간의 높이 충돌을 방지하는 강제 인라인 스타일을 적용하여 모든 디바이스에서 카드가 찌그러지지 않도록 고정.

### 👁️ 가독성 및 디자인 디테일 (Visual Readability)
- **Deep Gradient Overlay**: 사진 하단에 95% 농도의 그라데이션을 적용하여 밝은 사진 위에서도 흰색 글자가 선명하게 노출.
- **Drop Shadow**: 주요 텍스트와 가격, 영양 정보 레이어에 그림자 효과를 주어 배경으로부터 시각적 분리.
- **Dynamic Stamps**: 드래그 방향에 따라 실시간으로 나타나는 `LIKE` / `NOPE` 스탬프로 즉각적인 피드백 제공.
