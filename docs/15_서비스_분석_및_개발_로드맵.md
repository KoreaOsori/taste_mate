# 🍱 TasteMate 서비스 분석 및 개발 로드맵 (보고서 15)

본 문서는 TasteMate 프로젝트의 초기 기획부터 현재까지의 개발 과정을 돌아보고, 향후 서비스 고도화를 위한 구체적인 기술 로드맵을 제시합니다.

---

## 1. 개발 과정 회고 (Retrospective)

TasteMate는 "사용자의 신체 데이터와 위치 정보를 결합한 초개인화 식단 추천"이라는 핵심 목표 아래 세 가지 주요 마일스톤을 통과해 왔습니다.

### **Phase 1: 인프라 및 핵심 기능 구축 (기초 확립)**
*   **환경**: FastAPI(Back) + React(Front) + Supabase(DB/Auth)의 하이브리드 아키텍처 구축.
- **성과**: 기본적인 로그인/회원가입, 신체 정보 입력(BMR 계산), 대시보드 시각화 완료.
- **특이사항**: Docker 환경 도입으로 일관된 개발/배포 환경 확보.

### **Phase 2: 추천 알고리즘 및 데이터 연동 (지능화)**
*   **알고리즘**: 단순 필터링을 넘어 **RAG (Retrieval-Augmented Generation)** 패턴 도입.
*   **연동**: Kakao/Naver API 기반 위치 정보 취득 및 Reverse Geocoding 연동.
- **성과**: 외부 날씨, 현재 시간, 사용자의 감정 및 동행자 유무를 고려한 Re-ranking 엔진 구현.

### **Phase 3: 사용자 경험 최적화 (안정화)**
*   **영속성**: 새로고침 시 화면 리셋 현상 방지(Screen Persistence), 게스트 세션 유지 로직 구현.
*   **위치 동기화**: 실시간 좌표 취득 및 DB 프로필 자동 동기화 기능 고도화.
- **성과**: 하드코딩된 데이터 제거 및 실제 사용자 데이터 기반의 다이나믹 UI 완성.

---

## 2. 향후 개발 로드맵 (Future Roadmap)

현재의 안정된 기반을 바탕으로, 서비스의 상용화와 사용자 가치 극대화를 위한 다음 단계들입니다.

### **[Milestone 4] 기능 고도화 및 콘텐츠 보강 (Short-term)**
- **비전**: 사용자가 입력하지 않아도 자동으로 식단을 관리하는 서비스.
*   **AI 식단 기록 (Vision API)**: 음식 사진 촬영 시 자동으로 칼로리와 영양소를 분석하는 기능 (OCR & GPT-4o Vision).
*   **커뮤니티 활성화**: 커뮤니티 게시글 이미지 업로드, 좋아요/댓글 실시간 동기화 (Supabase Realtime 활용).
*   **푸시 알림 시스템**: 설정된 식사 시간에 맞춘 개인화 알림 및 건강 요약 리포트 전송.

### **[Milestone 5] 기술 부채 해결 및 성능 최적화 (Mid-term)**
- **비전**: 수많은 사용자가 동시에 사용해도 빠르고 안정적인 서비스.
*   **검색 성능 최적화**: 벡타 DB(Vector DB) 검색 고도화 및 캐싱 시스템(Redis 등) 도입.
*   **백엔드 모듈화**: 마이크로서비스 지향형 리팩토링 및 테스트 자동화 (Pytest) 강화.
*   **모바일 최적화**: PWA(Progressive Web App) 도입 또는 React Native를 통한 네이티브 앱 전환 고려.

### **[Milestone 6] 상용화 및 운영 체계 구축 (Long-term)**
- **비전**: 실제 시장에서 수익을 창출하고 지속 가능한 서비스.
*   **클라우드 배포**: AWS 또는 GCP 기반의 확장 가능한 인프라 배포 (Terraform/Kubernetes).
*   **SEO 및 분석**: 검색 엔진 최적화 및 사용자 행동 분석 툴(GA4, Amplitude) 연동.
*   **정기 리포트**: 주간/월간 단위의 건강 분석 유료 리포트 모델 도입.

---

## 3. 기술적 도전 과제 (Technical Challenges)

1.  **AI 분석의 정확도**: 비전 인식을 통한 영양 성분 추출 시 오차 확률을 줄이는 정교한 프롬프트 엔지니어링 필요.
2.  **데이터 무결성**: 로컬 스토리지와 실시간 DB 간의 동기화 충돌 방지를 위한 최적의 상태 관리 전략.
3.  **추천 정합성**: 사용자의 피드백을 추천 엔진에 다시 학습시키는 'Reinforcement Learning from User Feedback' 구조 설계.

---

TasteMate는 단순한 추천 앱에서 **"나보다 나를 더 잘 아는 AI 건강 파트너"**로 진화하고 있습니다.
