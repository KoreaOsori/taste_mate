# 10. DB 연동 및 로그인 로직 구축 과정 상세 보고서

본 문서는 TasteMate 프로젝트의 데이터베이스(DB) 연결 구조와 로그인 프로세스의 개선 과정을 기술하며, 발생했던 주요 기술적 난관과 그 해결 방안을 상세히 기록합니다.

## 1. 개요 및 기술 스택
TasteMate는 클라우드 기반의 Backend-as-a-Service(BaaS)인 **Supabase**를 데이터 인프라의 핵심으로 사용합니다.

- **Frontend**: React (Vite), Supabase JS SDK, Axios
- **Backend**: FastAPI (Python), Supabase Python SDK (`supabase-py`)
- **Database**: PostgreSQL (Supabase Managed), Supabase Auth

## 2. DB 및 SQL/Query 구축 과정
### 2.1 테이블 설계 및 연동
- **Profiles 테이블**: 사용자의 기본 정보(이름, 나이, 키, 몸무게) 및 목표(목표 칼로리, 목표 체중)를 저장합니다.
- **SQL 및 쿼리 최적화**: 
    - `user_id` (UUID)를 기본 키(PK)로 사용하여 Supabase Auth와 1:1 관계를 형성했습니다.
    - 데이터 일관성을 위해 `Upsert` 로직을 구현하여 프로필 정보를 생성하거나 업데이트할 때의 중복을 방지했습니다.
    - 백엔드(`profile.py`)에서는 `supabase-py`를 활용해 특정 UUID에 대한 고속 조회를 수행하도록 쿼리를 작성했습니다.

## 3. 로그인 로직 및 환경 개선
### 3.1 라우팅 및 세션 관리
- **문제점**: 기존에는 Supabase 세션이 남아있을 경우 무조건 홈 화면 진입을 시도했으나, DB에 프로필 정보가 없는 경우에도 홈으로 넘어가 화면이 깨지는 현상이 있었습니다.
- **해결책**: `App.tsx`의 `fetchUserProfile` 로직을 개선하여, 로그인 성공 시 즉시 백엔드 API를 통해 프로필 존재 여부를 확인합니다.
    - 데이터가 있을 경우: **메인 홈(대시보드)** 직행
    - 데이터가 없을 경우(최초 가입자): **온보딩(위치 권한/정보 입력)** 화면으로 유도

### 3.2 에러 메시지 세분화 및 한글화
- **기술적 도전**: 보안상 Supabase Auth는 아이디 부재와 비밀번호 틀림을 `Invalid login credentials`로 통합하여 반환합니다.
- **해결**: 백엔드에 `service_role` 권한을 가진 `/api/v1/auth/check-user` 엔드포인트를 신설했습니다.
    - 로그인 실패 시 프론트엔드에서 이 API를 호출하여 해당 이메일의 가입 여부를 먼저 체크합니다.
    - 이후 **"회원 정보가 존재하지 않습니다"**와 **"비밀번호가 틀렸습니다"**로 구분하여 사용자에게 한국어로 친절하게 안내합니다.

## 4. 주요 오류 및 해결 과정 (TROUBLESHOOTING)

| 발생 오류 | 원인 분석 | 해결 방법 |
| :--- | :--- | :--- |
| **CORS Policy Error** | 프론트엔드가 3000(또는 3001) 포트에서 실행될 때 백엔드(8000)에서 허용되지 않음 | `backend/main.py`의 `CORSMiddleware` 설정에 모든 예상 로컬 호스트(3000, 3001)를 명시적으로 등록함 |
| **Profile Not Found (404)** | 로그인 성공 후 DB에 프로필이 생성되기 전 정보를 요청하거나 유저 ID 매칭 실패 | 백엔드 API에서 UUID 데이터 타입을 문자열로 정확히 변환하여 쿼리하도록 수정 및 로그 추가 |
| **Network Error (Axios)** | 백엔드 서버 기동 시 특정 라이브러리(`pydantic`의 `EmailStr` 등) 충돌로 서버 다운 | 의존성을 보다 범용적인 데이터 타입(`str`)으로 변경하고 방어적 코딩 적용 |
| **Login Persistence Issue** | 로그아웃 기능 미비로 다른 계정 전환이 불가능함 | 온보딩 및 모든 세부 화면에 전역 `handleLogout` 기능을 통합하여 세션 완전 초기화 구현 |

## 5. 현재 구축된 로그인 아키텍처 흐름도
1.  **사용자 로그인 시도** (LoginScreen)
2.  **Supabase Auth 인증**
3.  **인증 성공 시**: `authUserId` 획득 후 백엔드 프로필 API 호출
    - 성공: 홈 화면으로 이동
    - 404 에러: 온보딩 프로세스 시작
4.  **인증 실패 시**: 백엔드 `/check-user` 호출
    - 사용자 있음: "비밀번호가 틀렸습니다" 표시
    - 사용자 없음: "회원 정보가 없습니다" 표시

이와 같은 체계적인 개선을 통해 TasteMate는 보안과 사용자 편의성을 동시에 갖춘 안정적인 로그인 및 데이터 관리 환경을 구축하였습니다.
