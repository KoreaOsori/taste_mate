# [보고서 19] 상세 정보 UI 개선 및 고도화 (스크레이핑 & 지도 통합)

본 보고서는 추천된 식당의 상세 정보를 사용자에게 더 정확하고 풍부하게 제공하기 위해 진행된 팝업창 개선 및 데이터 수집 고도화 과정을 담고 있습니다.

---

## 🗺️ 1. 지도 서비스 통합 (Map Integration)

### A. 동적 카카오맵 렌더링
- **동적 로딩**: 페이지 로드 시가 아닌, 팝업이 열릴 때 지도를 동적으로 초기화하여 리소스 최적화.
- **에러 핸들링**: 카카오 API 호출 실패 시 '다시 시도' 버튼과 가이드 문구를 노출하여 사용자 불편 최소화.
- **마커 표시**: 사용자의 현재 위치(집 아이콘)와 식당 위치(핀 아이콘)를 한 화면에 동시 노출하여 거리감 체감 유도.

### B. 길찾기 기능 강화
- **카카오맵 연동**: 버튼 클릭 한 번으로 카카오맵 길찾기 모드(좌표 기반)로 즉시 연결.

---

## 🍱 2. 데이터 수집 고도화 (나노바나나 & Vision)

### A. 상세 정보 실시간 추출
- **문제**: 기존 Naver Search API는 영업시간과 상세 정보를 제한적으로만 제공함 (빈값인 경우가 많음).
- **해결 (나노바나나)**: 브라우저 기반 스크레이핑 기술을 도입하여 **네이버 플레이스**의 실제 상세 페이지 데이터를 직접 추출.
- **추출 데이터**:
    - **전화번호**: 실제 매장 관리 번호 (예: 055-294-3333).
    - **영업시간**: 월~일 단위의 상세 시간표, 브레이크타임, 정기 휴무일.

### B. UI 레이아웃 최적화
- **멀티라인 지원**: 추출된 주간 시간표를 `whitespace-pre-line` 속성을 통해 자연스러운 줄바꿈으로 노출.
- **링크 워핑 (Word-wrap)**: 매우 긴 네이버 지도 URL이 팝업 너비를 초과하지 않도록 `break-all` 스타일을 적용하여 시큐어한 레이아웃 유지.

---

## 🛠️ 3. 기술 스택 및 라이브러리

- **Backend**: Python (Scraper Logic), Naver Local API, Kakao Rest API.
- **Frontend**: React (Dialog), Kakao Maps SDK, Tailwind CSS.

---

> [!IMPORTANT]
> 본 고도화 작업의 핵심은 **"데이터의 실재성"**입니다. 사용자가 팝업을 열었을 때 실제 매장의 최신 영업 정보와 실제 이미지를 볼 수 있도록 하는 데 집중했습니다.
