# 스와이프 기반 음식 추천 UI 구현 보고서 (08)

사용자가 식당을 더 직관적이고 재미있게 선택할 수 있도록 Tinder 스타일의 스와이프 인터페이스를 도입했습니다.

## 1. 개요
기존의 버튼 클릭 또는 단순 캐러셀 방식에서 벗어나, 카드를 좌우로 밀어서(Swipe) '찜하기' 또는 '넘기기'를 결정하는 인터페이스를 구현했습니다. 시각적인 음식 사진을 전면에 내세워 사용자의 직관적인 메뉴 결정을 돕는 것이 핵심입니다.

## 2. 기술적 구현 (Frontend)

### 스와이프 엔진
- **framer-motion**: 리액트 애니메이션 라이브러리를 활용하여 부드러운 드래그 및 탈출(Exit) 애니메이션을 구현했습니다.
- **AnimatePresence**: 카드 교체 시 이전 카드가 자연스럽게 화면 밖으로 사라지고 새 카드가 나타나도록 연출했습니다.

### 컴포넌트 구조
- **RestaurantRecommendationCardView**: 
  - 카드 스택 및 드래그 로직 담당.
  - 하단 액션 버튼 (❌ 추천 제외, 🔄 다시 추천, ❤️ 찜하기) 배치.
  - 사진 클릭 시 상세 정보 모달 호출.

## 3. 주요 기능 및 UX

### Tinder 스타일 카드 스택
- **Right Swipe (또는 ❤️ 버튼)**: 맛집 찜하기. 카드가 오른쪽으로 날아가며 다음 추천으로 넘어갑니다.
- **Left Swipe (또는 ❌ 버튼)**: 이번엔 패스. 카드가 왼쪽으로 날아갑니다.
- **진행률 인디케이터**: 상단에 현재 몇 번째 추천을 보고 있는지 표시합니다.

### 이미지 중심 레이아웃
- 고화질 음식 사진을 카드 전체 배경으로 배치했습니다.
- 사진 하단에 식당 이름, 별점, 거리 등 핵심 정보만 그라데이션 오버레이로 표시하여 몰입감을 높였습니다.

### 상세 정보 모달 (Detailed Popup)
- 사진을 클릭하면 하단에서 부드럽게 상세 정보창이 올라옵니다.
- **상세 정보**: 주요 메뉴 가격, 상세 칼로리(kcal), 영양 성분 비율, 식당의 정확한 주소.
- **연동 기능**: [이걸로 먹으게요!] 버튼 클릭 시 네이버 지도 등 외부 맵과 연동되어 바로 길찾기가 가능합니다.

## 4. 기대 효과
- **추천 신뢰도 향상**: 복잡한 텍스트 대신 음식 사진을 먼저 보여줌으로써 사용자의 메뉴 선택 속도를 높입니다.
- **사용자 참여도**: 게임 같은 스와이프 동작을 통해 서비스를 더 자주 이용하게 만드는 '게이미피케이션' 요소를 추가했습니다.
